About Me

Greetings! I'm Peter, a tech enthusiast with a lifelong passion for all things digital. My journey into the world of Information Technology started at a young age, fueled by my curiosity for computers. I would eagerly disassemble and reassemble old computers from my dad's office, laying the foundation for my hands-on approach to technology.

As I grew up and built on my technical skills I began modding games I enjoyed playing which allowed me to get exposure to a variety of languages. The ability to create and experience my additions firsthand in projects I loved further grew my interest in programming. Upon reaching high school, I took the next step by developing a python text-based RPG game 'Dragon Slayer' along with a 2D Pokemon game with some Java.

Beyond the screen, I'm equally passionate about the great outdoors, embarking on annual adventures such as portaging the Bowron Lakes or torturing my cardiovascular system with weeklong backpacking trips. I also developed a keen interest in aviation, I quickly found myself flying airplanes before even learning to drive and I now have my own private pilot's license!

I've just graduated from BCIT's Computer Information Technology program and I'm excited to begin working in the industry.

Previous Projects

Terraform-Ansible Project

Automated deployment for a simple front-end web application which fetched database information through a seperate backend instance. All public requests are routed through nginx on the frontend and all internal queries for the database are processed through the backend.

The servers run with Python on EC2 Instances and the database is hosted on an RDS instance all automated with Terraform and Ansible. We store the terraform state in an S3 Bucket to ensure it stays consistent with the actual state of terraform. I also wrote a simple bash script to run all of the commands in order and remove any previous working instances ran locally.

Acmestock-Co

Acmestock Co was a scaled microservice based application which was hosted on Azure cloud VMs had several endpoints that could be requested (some with parameters) to query data from the application's database. My model was a stock trading company, and had endpoints to create new sell orders and company listings. I set up and configured kafka/zookeeper message broker service to ensure consistency among internal requests.

The entire system was containerized through docker and some of the services such as the receiver and storage service were scaled up to account for large amounts of traffic during testing. I also implemented a health check endpoint which would return 200 for purposes of automation (Restarts the docker container if the application doesnt send a 200 response).
